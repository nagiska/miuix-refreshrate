# Miuix Refresh Rate

一款面向已 Root 的 Android 设备开发的刷新率控制工具，使用 Miuix 风格界面，支持手动切换显示模式以及按照前台应用自动应用不同的分辨率和刷新率。

> 项目仍处于开发和设备适配阶段。刷新率切换依赖系统、SurfaceFlinger 和厂商实现，不保证适用于所有设备。

## 功能

- 查看设备支持的分辨率和刷新率模式
- 手动切换指定显示模式
- 为不同应用单独配置分辨率和刷新率
- 进入已配置应用时自动升档或降档
- 离开已配置应用后恢复进入前的显示模式
- 分阶段切换高刷新率，降低直接跨档失败的概率
- 对切换结果进行 Android API、系统设置和 SurfaceFlinger 状态校验
- 恢复后延迟复检，检测刷新率反弹并重新应用目标模式
- 前台常驻通知展示当前应用和刷新率状态
- 查看、清空和导出详细运行日志
- 支持识别部分系统分身应用配置

## 使用要求

- Android 13 或更高版本（`minSdk 33`）
- 设备已经 Root
- Root 管理器允许本应用执行超级用户命令
- 开启本应用的无障碍服务
- 建议关闭系统对本应用的省电限制，并允许后台运行

本项目主要针对 MIUI/HyperOS 设备开发。其他 Android 系统能否正常使用取决于设备是否支持项目使用的显示控制命令。

## 使用方法

1. 安装 APK 并授予 Root 权限。
2. 打开“设置”页面，进入系统无障碍设置。
3. 开启“屏幕刷新率”的无障碍服务。
4. 打开“应用”页面，选择需要单独控制刷新率的应用。
5. 开启“自定义刷新率”，选择目标分辨率和刷新率。
6. 切换到目标应用，观察是否自动应用配置。
7. 离开目标应用后，程序会尝试恢复进入应用前的显示模式。

首页也可以直接点击设备支持的刷新率档位进行手动切换。

## 自动切换原理

应用通过无障碍服务和前台窗口轮询识别当前应用，并使用 Root 权限执行以下类型的操作：

- 设置系统最高、最低和用户刷新率
- 设置 MIUI 刷新率配置
- 设置用户首选显示模式
- 调用 SurfaceFlinger 切换显示模式

高刷新率切换可能会按照设备支持的模式逐档执行。例如从 120Hz 切换到 165Hz 时，可能依次经过 132Hz、144Hz、156Hz 和 165Hz。

恢复显示模式后，程序还会继续进行多次延迟复检。如果检测到系统重新切回了其他刷新率，会再次强制应用恢复目标。

## 运行日志

在“设置 -> 运行日志”中可以查看或导出日志。日志包含：

- 无障碍服务连接、中断和销毁状态
- 当前识别到的前台应用
- 应用配置匹配结果
- 升档、降档和逐档切换过程
- Root 命令退出状态和错误信息
- Android API 报告的刷新率
- `dumpsys display` 活动模式
- SurfaceFlinger 帧周期推算的实际刷新率
- 恢复后的 `POST_RESTORE` 延迟复检结果

反馈自动切换问题时，建议先清空旧日志，然后完整执行一次以下流程：

1. 停留在普通应用或桌面。
2. 进入已配置刷新率的应用。
3. 等待切换完成。
4. 返回桌面或另一个未配置应用。
5. 等待至少 12 秒。
6. 导出并附上完整日志。

重点日志字段：

- `physical`：根据 SurfaceFlinger 帧周期推算的刷新率
- `active`：系统显示服务报告的活动模式刷新率
- `preferred`：用户首选显示模式刷新率
- `peak/min/user/miui`：相关系统设置值
- `POST_RESTORE`：退出应用后的延迟复检和重新应用记录

## 构建

项目使用 Kotlin Multiplatform、Compose Multiplatform 和 Miuix 构建。

环境要求：

- JDK 17
- Android SDK
- 项目所需的 `local.properties`

构建 Debug APK：

```bash
./gradlew :composeApp:assembleDebug
```

输出位置：

```text
composeApp/build/outputs/apk/debug/composeApp-debug.apk
```

仓库的 GitHub Actions 会在推送到 `main`、创建 Pull Request 或手动触发时构建 Debug APK，并将其上传为 `debug-apk` Artifact。

## 项目结构

```text
composeApp/src/commonMain/       通用 Compose 界面和数据模型
composeApp/src/androidMain/      Android 平台实现、Root 工具和无障碍服务
.github/workflows/build.yml      GitHub Actions 构建配置
```

主要实现：

- `KeepAliveAccessibilityService.kt`：前台应用识别和逐应用自动切换
- `RootUtils.kt`：Root 命令、显示模式切换和状态解析
- `AutoOverclockManager.kt`：显示模式扫描和当前模式读取
- `RuntimeLog.kt`：运行日志保存与读取

## 注意事项

- 超出设备官方规格的刷新率可能导致黑屏、闪屏、触控异常、发热或功耗增加。
- 不正确的显示模式可能导致界面暂时不可操作，请确保拥有其他恢复显示设置的方法。
- 系统省电策略、温控策略和厂商刷新率服务可能覆盖本应用的设置。
- 某些设备虽然在系统状态中显示目标刷新率，面板实际输出仍可能不同，应结合 `physical` 日志判断。
- 使用本项目产生的设备风险由使用者自行承担。

## 隐私

应用配置和运行日志保存在设备本地。导出日志时可能包含应用包名、显示模式、Root 命令结果和设备显示状态，请在公开分享前检查内容。
